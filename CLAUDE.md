# dawn

A Next.js application. Use this file as the source of truth for the tech stack
and conventions; defer to it before reaching for general defaults.

## Stack

- **Runtime / framework**: Next.js (App Router, RSC, Turbopack), React 19
- **Language**: TypeScript with strict + `noUncheckedIndexedAccess`
- **Package manager**: `pnpm` (only ‚Äî never `npm`/`yarn`)
- **Styling**: Tailwind CSS v4 (`@import "tailwindcss"`, no `tailwind.config.*`)
- **Components**: ShadCN UI (`new-york` style, `neutral` base, lucide icons)
- **Database**: Supabase (Postgres + Auth)
  - Server clients in `src/lib/supabase/{server,client,middleware}.ts`
  - Auth session refresh runs in `src/middleware.ts`
  - The trust boundary is `/api/chat`: it validates the user, then the Inngest
    function uses a service-role Kysely client (`@/lib/db/admin`) to write
    deltas, bypassing RLS.
- **Query builder**: [Kysely](https://kysely.dev) over the Supabase Postgres
  connection (`pg` driver). The DB type comes from
  `kysely-supabase`'s `KyselifyDatabase<>` applied to the generated
  Supabase types ‚Äî see `src/lib/db/types.ts`.
- **Background jobs**: Inngest. Functions live in `src/lib/inngest/functions.ts`
  and are served at `/api/inngest`.
- **AI**: Vercel `ai` + `@ai-sdk/anthropic`, calling `claude-sonnet-4-6`.
  Sample chat at `/chat` (under `(authed)/`) calling `/api/chat`. Streaming
  runs inside an Inngest function (`chat-stream`); the route subscribes to a
  per-stream realtime channel and forwards SSE.
- **Lint/format**: Biome (do **not** add ESLint or Prettier).
- **Tests**: Vitest + Testing Library + happy-dom.
- **Env validation**: Zod schema in `src/lib/env.ts`.

## Layout

```
src/
  app/                  # Next.js App Router
    (authed)/           # Auth-gated route group
      layout.tsx        # Auth gate (redirects to /login if no session)
      chat/             # Chat page + chat-view client component
    api/chat/           # /api/chat ‚Äî validates user, kicks off Inngest stream
    api/inngest/        # Inngest serve handler
    auth/callback/      # Supabase OAuth callback (exchanges code for session)
    login/              # Login page + server actions
  components/ui/        # ShadCN-generated components
  lib/
    db/                 # Kysely + Supabase-typed schema
      database.types.ts # Generated by `pnpm generate` (do not edit)
      types.ts          # KyselifyDatabase<Database>
      index.ts          # Kysely client (server-only, user-context via Supabase)
      admin.ts          # Service-role Kysely client (server-only, bypasses RLS)
    env.ts              # Zod env parsing
    inngest/            # client + functions
    supabase/           # browser/server/middleware clients
    utils.ts            # `cn()`
  middleware.ts         # Supabase auth session refresh
supabase/
  migrations/           # Schema source of truth (applied by `pnpm db:reset`)
scripts/
  generate.mjs          # All codegen ‚Äî runs as `predev`/`prebuild`
```

## One command to rule dev

`pnpm dev` does everything needed to bring the app up:

1. `predev` runs `scripts/generate.mjs` ‚Äî currently regenerates
   `src/lib/db/database.types.ts` from the local Supabase schema (no-op if
   Supabase isn't initialized/running, so it never blocks dev).
2. `concurrently` launches `next dev --turbopack` and the Inngest dev server
   pointed at `/api/inngest`.

Add new codegen steps to `scripts/generate.mjs` rather than to `package.json` ‚Äî
"one script does everything" is a hard requirement.

## Environment

Copy `.env.example` to `.env.local` and fill it in. Required for full
functionality:

- `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_DB_URL` (Kysely)
- `SUPABASE_SERVICE_ROLE_KEY` (Inngest function bypasses RLS via `lib/db/admin`)
- `ANTHROPIC_API_KEY` (chat route)
- `SUPABASE_AUTH_GITHUB_CLIENT_ID` + `SUPABASE_AUTH_GITHUB_SECRET` for OAuth
  login (GitHub is the only supported OAuth provider)
- `INNGEST_DEV=1` for local dev (points the SDK at the local dev server)
- `DEV_AUTH_ENABLED=1` (optional) ‚Äî exposes a local-only email sign-in at
  `/api/dev/login` and a "Local dev sign-in" form on `/login`. Never set in
  deployed environments.

`pnpm db:start` prints fresh values for the Supabase keys.

## Conventions

### TypeScript

- **Strict TS** ‚Äî fix the type, don't `// @ts-ignore`. `any` is a `warn`, not
  a tool.
- **`noUncheckedIndexedAccess: true`** ‚Äî array/record access is `T | undefined`.
  Narrow it; don't paper over it.
- **No unchecked casts** ‚Äî `as Foo` on values of unknown shape is a bug
  waiting to happen. Validate at the boundary instead (Zod, type guards).
- **Explicit casts are a code smell** ‚Äî every `as` (and every `!` non-null
  assertion) needs a one-line comment explaining *why* it's safe (an
  invariant the type system can't see, a generated-type quirk, etc.). No
  comment, no cast.
- **All external interfaces are validated** ‚Äî anything entering the system
  (HTTP bodies, query params, third-party API responses, env vars, message
  payloads) goes through a Zod schema (or equivalent runtime validator)
  before it's typed. Trusting an `unknown` because TypeScript stopped
  complaining doesn't count.

### Linting

- **No ESLint/Prettier** ‚Äî Biome is the only formatter/linter. CI runs
  `pnpm lint` + `pnpm typecheck`.
- **`noFloatingPromises` must stay green** ‚Äî every promise is `await`ed,
  returned, or explicitly handled. Fire-and-forget belongs in an Inngest
  function, not a `void` cast.
- **`noImportCycles` must stay green** ‚Äî break cycles by extracting shared
  types/utilities, not by suppressing the rule.
- **Pre-push checklist** ‚Äî run `pnpm typecheck` and `pnpm lint` locally
  before pushing. CI failing on these is avoidable noise.

### Architecture

- **Leverage the framework, don't reinvent it** ‚Äî Next.js, Supabase, Inngest,
  Vercel `ai`, Zod, Kysely, ShadCN are already pulled in. Reach for the
  built-in primitive (route handlers, server components, Inngest steps,
  Supabase auth, Zod parsing) before writing your own.
- **No packaged ORMs** ‚Äî Kysely is a query builder, not an ORM, and that's
  deliberate. We need to drop down to raw SQL without fighting an
  abstraction. Don't add Prisma, Drizzle, TypeORM, etc.
- **Server vs client Supabase** ‚Äî never import `lib/supabase/server` or
  `lib/db` from a `'use client'` file.
- **Generated files** ‚Äî `database.types.ts` is regenerated; never hand-edit.
- **Style additions** ‚Äî prefer `cn()` + Tailwind utilities; add ShadCN
  primitives via `pnpm dlx shadcn@latest add <component>`.
- **Background work** ‚Äî anything async/durable belongs in an Inngest function,
  not a fire-and-forget API route.
- **DB writes** ‚Äî user-context reads use the Supabase server client (PostgREST,
  RLS-honoring). Privileged writes use `@/lib/db/admin` ONLY from server code
  that has already validated user ownership upstream (currently: the Inngest
  `chatStream` function, gated by `/api/chat`).
- **Client ‚Üí server interactions** ‚Äî REST under `/api/...` is the preferred
  approach for **all future work**. Use Route Handlers for mutations
  (`POST` / `PUT` / `PATCH` / `DELETE`) and call them with `fetch` +
  `useTransition`. **Server actions are not used in this project.**
  (`src/app/login/actions.ts` predates this convention and will be migrated
  in a separate ticket.)
